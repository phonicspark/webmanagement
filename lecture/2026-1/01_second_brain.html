AI 리터러시 스쿨 1회차

카파시의 LLM Wiki로 나만의 AI 세컨드 브레인 만들기

RAG처럼 매번 다시 찾는 지식관리에서 벗어나, AI가 읽고 갱신하는 개인 위키를 만듭니다.

핵심 메시지

RAG는 매번 검색해서 답을 만듭니다. LLM Wiki는 지식이 쌓이기 때문에 시간이 지날수록 답이 더 좋아집니다.

개인 지식관리에는 단순 검색보다 축적이 필요합니다. 자료를 많이 저장하는 것보다 내 목적에 맞게 모으고, AI가 읽을 수 있는 위키로 컴파일하고, 다시 질문과 결과물로 연결하는 것이 중요합니다.

Gold In, Gold Out. 목적 있는 수집만이 좋은 데이터가 됩니다.

오늘 만들 것

Karpathy의 LLM Wiki 패턴을 바탕으로 Claude Code, Obsidian, Obsidian Web Clipper, Graphify를 연결한 개인 AI 세컨드 브레인 흐름을 설계합니다.

raw/ 수집한 원본을 수정하지 않고 보관하는 불변 자료실
wiki/ AI가 요약, 연결, 업데이트하는 컴파일된 지식 공간
CLAUDE.md AI가 매번 읽는 나의 설명서와 위키 운영 규칙
Graphify 위키 문서를 그래프로 바꿔 관계 기반 질의를 돕는 도구

사전 준비

먼저 Obsidian 새 볼트, Claude Code 연결, Obsidian Web Clipper, Python 3 환경의 Graphify를 준비합니다.

  • Obsidian
  • Claude Code
  • Obsidian Web Clipper
  • pip install graphifyy

숙제 3가지

  1. 나는 누구인가: 이름, 하는 일, 역할
  2. 왜 기록하고 싶은가: 지금 무엇이 안 되는지, 기록이 되면 무엇이 달라지는지
  3. 어떤 아웃풋을 만들고 싶은가: 누구를 위해, 어떤 형태로 만들고 싶은지

LLM Wiki 핵심 아이디어

일반적인 RAG는 질문할 때마다 관련 조각을 찾아 답을 만듭니다. LLM Wiki는 원본 자료를 `raw/`에 보관하고, AI가 그 자료를 읽어 `wiki/` 안의 마크다운 문서로 계속 정리하고 연결합니다.

새 자료가 들어오면 AI는 기존 개념 페이지, 엔티티 페이지, 요약 페이지, 인덱스, 로그를 업데이트합니다. Obsidian은 IDE, LLM은 프로그래머, wiki는 코드베이스처럼 작동합니다.

Step 1. 맥락 인터뷰

AI가 내 목적을 이해하도록 `나의 핵심 맥락.md`를 읽고 한 번에 하나씩 질문하게 합니다.

우리는 지금 "AI를 위한 세컨드 브레인"을 만들고 있어.

목적: 세상에 있는 유용한 정보와 지식을 체계적으로 모으고,
나의 맥락에 맞게 정리·연결해서, AI가 이를 읽고 활용할 수 있게 만드는 것.

'나의 핵심 맥락.md'는 내가 사전에 작성한 자기소개야.
이걸 읽고, 나의 맥락을 더 깊이 이해하기 위해 인터뷰해줘.

아래 순서로 한 번에 하나씩 질문해줘:
1. "나는 누구인가"를 더 깊이: 역할, 강점, 가치관
2. "왜 기록하고 싶은가"를 더 깊이: 지금 안 되는 것, 비전
3. "어떤 아웃풋을 만들고 싶은가"를 더 깊이: 대상, 형태, 1년 후 이상적인 모습

Step 2. CLAUDE.md 생성

인터뷰 결과를 바탕으로 AI가 매번 읽는 나의 설명서를 만듭니다.

내 볼트에 "나의 핵심 맥락.md" 파일이 있어.
이 파일을 읽고, CLAUDE.md를 만들어줘.

포함할 섹션:
1. "## 나는 누구인가" → 이름, 하는 일, 핵심 가치
2. "## 나의 역할들" → 각 역할별: 하는 일, 주요 관심사
3. "## 나의 비전과 목표" → 이루고자 하는 것, 타겟 독자/고객
4. "## AI에게 기대하는 것" → 어떤 도움을 원하는지
5. "## 작업 규칙" → 톤, 언어, 결과물 형태

Step 3. LLM Wiki 세팅

Karpathy의 LLM Wiki Idea File을 전달하고 내 목적에 맞는 구조와 운영 규칙을 만들게 합니다.

볼트/
├── CLAUDE.md
├── raw/
│   ├── articles/
│   ├── videos/
│   ├── podcasts/
│   ├── books/
│   └── CLAUDE.md
├── wiki/
│   ├── index.md
│   ├── log.md
│   └── CLAUDE.md
└── Output/
    └── CLAUDE.md
  • `raw/`는 절대 수정하지 않는 불변 원본입니다.
  • wiki 변경 시 `index.md`와 `log.md`를 반드시 업데이트합니다.
  • 내부 참조는 wikilink 형식, wiki 페이지는 YAML frontmatter를 사용합니다.
  • 질문할 때는 `index.md`를 먼저 보고, `raw/`는 마지막 수단으로 봅니다.

Step 4. 웹 클리퍼 템플릿

Obsidian Web Clipper 기본 템플릿을 내보낸 뒤, `raw/` 구조에 맞는 JSON 템플릿을 만듭니다.

지금 보고 있는 파일은 옵시디언 웹 클리퍼 기본 템플릿인데,
내 raw/ 폴더에 맞는 템플릿을 만들어줘.

이 JSON 파일을 여러 가지로 만들어줬으면 좋겠어:
- 아티클 (Article)
- 유튜브 (YouTube)
- 팟캐스트 (Podcast)
- 책 (Book)
- 연구 자료 (Research)

Step 5. 인제스트

모으기만 하면 지식이 아니라 저장소입니다. 왜 수집했는지, 지금 하는 일과 어떻게 연결되는지, 무엇을 해보고 싶은지를 답해야 내 지식이 됩니다.

방금 웹 클리퍼로 raw/에 저장한 글이 있어. 인제스트해줘.

1. 해당 파일을 읽어줘. raw/는 절대 수정하지 마.
2. 소스 요약을 만들어줘:
   - 핵심 주장 (Key Claims)
   - 언급된 엔티티 (사람, 도구, 조직)
   - 다루는 개념 (Concepts)
3. 나에게 질문해줘:
   - 이 글을 왜 캡처했어?
   - 지금 하고 있는 일과 어떻게 연결돼?
   - 이걸로 뭘 해보고 싶어?
4. 내 답변을 받으면 wiki/에 반영해줘.
raw/ 폴더를 스캔해서 아직 인제스트 안 된 파일을 모두 처리해줘.
대화 없이 바로 wiki/에 반영하고, 끝나면 요약을 보여줘.

Step 6. 스킬 만들기

2~3번 수동으로 해보고 패턴이 잡히면 `/ingest`, `/query`, `/lint` 스킬로 반복 작업을 자동화합니다.

  • `/ingest`: 새 raw 파일을 확인하고 wiki에 반영
  • `/query`: wiki 문서를 기반으로 질문에 답변
  • `/lint`: 깨진 링크, 오래된 주장, 고립 문서, 누락 연결을 정리

Step 7. Graphify

위키가 커지면 Graphify로 지식을 그래프 구조로 바꿔 관계 기반으로 탐색합니다.

pip install graphifyy
/graphify wiki/
/graphify query "브레인 트리니티와 LLM Wiki의 관계는?"
/graphify wiki/ --update
  • `graphify-out/graph.json`: AI가 참조하는 그래프 데이터
  • `graphify-out/graph.html`: 브라우저 시각화
  • `graphify-out/GRAPH_REPORT.md`: 그래프 분석 리포트
